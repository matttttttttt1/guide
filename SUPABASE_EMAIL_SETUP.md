# Supabase 이메일 인증 설정

## 문제 상황

회원가입 후 이메일 인증을 해야만 로그인이 가능합니다.
→ 회원가입 → 이메일 확인 → 인증 링크 클릭 → 로그인 → 프로필 완성

이것을 다음과 같이 변경하고 싶습니다:
→ 회원가입 → **바로 프로필 완성** (이메일 인증 없이)

## 해결 방법: 이메일 인증 비활성화

### 1. Supabase Dashboard 접속

https://supabase.com/dashboard/project/zrvqhppvpinjcmjouvdh

### 2. 이메일 인증 비활성화

1. 왼쪽 메뉴 → **Authentication** → **Providers**
2. **Email** 클릭
3. **Confirm email** 토글을 **OFF**로 변경
4. **Save** 클릭

### 3. 완료!

이제 회원가입하면:
1. 이메일 + 비밀번호 입력
2. **즉시 로그인 상태**
3. 프로필 완성 페이지로 자동 이동
4. 여행사명, 사업자번호 입력
5. 가이드 목록 페이지로 이동

## 프로덕션 환경에서는?

실제 서비스에서는 이메일 인증을 **활성화**하는 것을 권장합니다:
- 스팸 방지
- 유효한 이메일 확인
- 보안 강화

하지만 개발/테스트 환경에서는 비활성화하는 것이 편리합니다.

## 이메일 인증을 유지하면서 UX 개선하기

이메일 인증을 유지하고 싶다면:

1. **회원가입 페이지에 안내 추가**:
   ```
   "회원가입 후 이메일로 인증 링크가 전송됩니다.
    이메일을 확인하고 인증을 완료해주세요."
   ```

2. **인증 대기 페이지 생성**:
   ```
   회원가입 → 이메일 인증 대기 페이지
   → "이메일을 확인해주세요" 메시지
   → 이메일 인증 완료 → 프로필 완성 페이지
   ```

## 현재 설정 (권장)

**개발 환경**: 이메일 인증 **OFF**
- 빠른 테스트
- 즉시 사용 가능

**프로덕션**: 이메일 인증 **ON**
- 보안 강화
- 유효한 사용자만 가입
